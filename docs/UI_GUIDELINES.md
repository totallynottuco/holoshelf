# Holoshelf UI Guidelines

Holoshelf should default to compact, concise desktop UI.

- Prefer dense but readable layouts over large panels, oversized buttons, and decorative spacing.
- Keep controls visually integrated with their surrounding surface; avoid button-inside-button, box-inside-box, and stacked-border effects.
- Reuse one shared pattern for repeated controls such as dropdowns, row actions, popovers, accordions, and list rows.
- Keep labels short. Use icons, tint, or subtle state changes when text adds clutter.
- Align text left in lists and menus. Counts and status symbols should align consistently on the right.
- Let menus size to their content with a sensible maximum width; truncate only when content is genuinely too long for the available space.
- Use compact row heights, small gaps, and subtle dividers. Empty space should be intentional, not accidental.
- When a UI state is repeated in multiple places, fix the shared pattern instead of patching each instance differently.
- Playlist and rating pickers should use the shared compact popover patterns consistently across the profile shelf, player view, row actions, and mini-player.
